本文件用于约束零维团队项目中的安全开发和安全审查。具体技术栈实现仍需同时遵守 Spring Boot、FastAPI、Soybean Admin、数据库和依赖管理规范。

## 适用范围

- 登录、认证、鉴权、权限点、角色、租户和数据归属。
- 密码、Token、密钥、验证码、AK/SK、证书和连接串。
- 文件上传、下载、预览、导入、导出。
- 外部 URL、回调地址、Webhook、第三方接口。
- SQL、命令、路径、模板、表达式、反序列化等注入风险。
- 日志、审计、脱敏、异常返回和数据导出。

## 基本原则

- 后端必须默认不信任前端输入。
- 权限校验必须在服务端完成，前端隐藏按钮不能作为安全边界。
- 安全规则优先于开发便利和临时绕过。
- 敏感信息默认不输出、不落日志、不返回前端、不提交仓库。
- 安全相关变更必须说明影响范围、失败场景和回滚方案。

## 认证与会话

- 当前用户身份必须来自可信认证上下文，不从普通请求参数中信任用户 ID、角色、权限或租户 ID。
- Token、Session、Cookie 的过期、刷新、注销和失效策略必须清晰。
- 管理端接口必须要求认证，公开接口必须明确说明公开原因。
- 密码必须使用成熟哈希算法存储，禁止明文、可逆加密或弱哈希。
- 验证码、一次性 token、重置密码链接必须有有效期和使用次数限制。

## 鉴权与数据归属

- 查询、修改、删除、导出资源时必须校验数据归属。
- 角色权限、按钮权限、菜单权限不能代替数据级权限。
- 多租户、组织、部门、用户维度的数据必须明确访问范围。
- 批量操作必须逐项或按可信范围校验权限，禁止只校验入口参数。
- 越权失败应返回稳定业务错误，不暴露资源是否存在的敏感细节。

## 敏感信息

- 密码、Token、密钥、验证码、证书、银行卡、身份证、完整手机号等不得明文输出到日志、异常、响应或提交记录。
- 响应对象必须过滤密码、盐值、密钥、内部权限表达式和内部调试字段。
- 日志中需要定位问题时，只记录脱敏后的标识和必要上下文。
- 配置文件不得提交真实生产账号、密码、连接串和密钥。
- 测试数据和初始化数据不得使用真实生产敏感信息。

## 输入校验与注入防护

- 所有外部输入必须做类型、长度、格式、枚举值和业务范围校验。
- SQL 必须参数化，禁止字符串拼接用户输入。
- 排序字段、查询字段、导出字段必须使用白名单。
- 命令执行、模板渲染、表达式计算、反序列化等高风险能力必须避免接收未可信输入。
- 文件路径、对象存储 key、回调地址、跳转地址必须做合法性校验，防止路径穿越和 SSRF。

## 文件与外部资源

- 文件上传必须限制大小、后缀、MIME 类型和存储路径。
- 上传文件名必须重新生成，禁止直接使用用户输入作为服务器路径。
- 私有文件下载、预览和导出必须鉴权。
- 导入任务必须校验文件结构、行数、字段长度和枚举值。
- 外部 URL 请求必须设置超时，并限制协议、域名或网络范围。

## Web 安全

- 生产 CORS 必须限制可信来源，禁止无条件使用 `*`。
- 管理端敏感操作应考虑 CSRF、重复提交和幂等风险。
- 前端渲染富文本、HTML、Markdown 时必须考虑 XSS，禁止直接渲染未净化内容。
- Cookie 应按项目需要设置 `HttpOnly`、`Secure`、`SameSite`。
- CSP、下载文件类型和 iframe 嵌入策略按业务风险评估配置。

## 第三方回调与 Webhook

- Webhook 必须校验签名、时间戳、随机数或幂等号。
- 回调处理必须具备幂等能力，避免重复支付、重复发货、重复通知。
- 第三方原始报文如需保存，必须评估脱敏、加密和访问权限。
- 第三方调用失败不能把原始异常、密钥、完整报文返回给前端。

## 日志与审计

- 登录、权限变更、角色变更、导出、删除、批量修改等高风险操作应记录审计信息。
- 审计日志应包含操作人、对象、结果、时间、来源和必要业务 ID。
- 未知异常记录服务端堆栈，但对外返回受控错误信息。
- 不重复打印同一异常，避免日志噪声掩盖真实风险。

## 安全交付清单

- 是否校验认证、鉴权和数据归属。
- 是否避免信任前端传入身份、角色、租户或权限。
- 是否过滤响应和日志中的敏感信息。
- 是否使用参数化查询和白名单排序。
- 是否校验文件上传、下载、导入、导出风险。
- 是否设置第三方调用超时、签名校验和幂等处理。
- 是否限制生产 CORS、Cookie 和前端富文本安全风险。
- 是否补充安全相关测试或说明无法测试的原因。
